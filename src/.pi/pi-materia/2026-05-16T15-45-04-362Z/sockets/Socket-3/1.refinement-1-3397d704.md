I understand the enhancement as: broaden Materia’s existing bounded “same-socket/same-node” recovery so that more failures are retried automatically, instead of requiring a manual `/materia recast`. Current code only auto-recovers context-window failures; generic provider/agent failures fail fast even though recast often succeeds.

Repo context I found:
- Recovery policy/workflow lives in `src/application/recoveryPolicy.ts` and `src/application/recoveryWorkflow.ts`.
- Native lifecycle wires it in `src/runtime/nativeLifecycle.ts`.
- Existing tests cover context-window recovery, transient WebSocket preservation, recast/revive, and classifications.

Proposed plan:
1. Generalize recovery reasons
   - Add a generic recoverable reason, e.g. `turn_failure` / `generic_turn_failure`.
   - Keep `context_window` special because it should run compaction before retry.
   - Keep plain transient WebSocket failures on the current “preserve awaiting, don’t retry” path.

2. Reuse the existing bounded same-socket retry workflow
   - For generic recoverable failures, skip compaction and resend the active prompt.
   - Preserve existing attempt limits, recovery identity keys, exhaustion metadata, `/materia revive`, and event structure where possible.
   - Update exhaustion reason types and cast-state typing.

3. Broaden lifecycle integration
   - Apply generic same-socket recovery to assistant error entries and agent-end failures with no assistant output.
   - Consider also applying it to safe post-response processing failures such as invalid JSON / handoff validation, since those are likely manual-recast-recoverable.
   - Avoid retrying utility socket failures or errors after non-idempotent state advancement unless we add explicit rollback/snapshot protection.

4. Tests and docs
   - Add unit tests for generic recovery classification/workflow.
   - Add native tests proving generic provider-ish failures retry once and then exhaust.
   - Add tests that context-window recovery still compacts, transient WebSockets still preserve awaiting state, and manual recast/revive behavior remains intact.
   - Update README/docs language from “context-window same-socket recovery” to “bounded same-socket recovery, with compaction for context-window failures.”
